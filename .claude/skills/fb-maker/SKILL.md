# fb-maker：Newsletter → Facebook 引流帖

## 输入

用户提供 newsletter 文章原文（完整或节选均可）。

---

## 改写流程

### 第一步：找核心洞见

找出文章真正想说的核心判断——通常藏在中间或结尾。把它挪到帖子最前面，作为整篇的阅读框架。

> 判断标准：如果读者第一遍读完需要回头才能抓住脉络，说明核心洞见位置不对。

### 第二步：三段式结构重组

- **开头**：核心洞见或反直觉观点，制造「咦，为什么」的好奇心
- **中间**：具体经历或论据，支撑核心洞见
- **结尾**：一句话点出启示，接 callback

### 第三步：写结尾 callback

一句话引导读者前往 newsletter，规则如下：

**内容要求：**

- 点名「我的 newsletter」
- 回扣文章中的具体概念（不能泛泛说「详细内容」）
- 预留链接占位符：`[我的 newsletter](链接)`

**句式轮换（每篇帖子使用不同句式）：**

| 句式    | 模板                                      |
| ----- | --------------------------------------- |
| 破折号式  | `具体内容——我在[我的 newsletter](链接) 里详细写了。`    |
| 直接陈述式 | `这个问题我在[我的 newsletter](链接) 里写清楚了：具体内容。` |
| 邀请式   | `如果你想知道具体内容，可以看[我的 newsletter](链接)。`    |

记录当前对话已用过的句式，下一篇自动切换。

### 第四步：写三个标题选项

- **反直觉悬念型**：制造「为什么」的好奇心，点击欲最强
- **核心判断型**：直接点出文章最重要的洞见
- **读者处境型**：说出读者正在经历的困境，让人感到「说的就是我」

---

## 改写原则

- **严格保持原文书写风格**，不改变语言习惯
- **不自我发挥**，不添加原文没有的观点
- **内容必须完整**，帖子本身能独立给读者提供价值，不依赖 newsletter 才能读懂
- 不用 AI 腔：不用「总结来说」「值得注意的是」「不得不说」
- 不用破折号 —— 作为强调手段（是 AI 生成的信号）

---

## 输出格式

每次输出顺序：

1. **三个标题选项**（放在最上面，方便先选标题）
2. **完整改写帖子**（Markdown 格式，方便直接复制粘贴）

接受修改指令（如「把 XX 那段拿掉」「换成读者处境型标题」「压缩到 200 字以内」）后直接更新输出。
